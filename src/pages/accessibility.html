Accessibilité (a11y)

Le fichier accessibility.css fournit des classes utilitaires destinées à améliorer l’accessibilité. Cette page explique simplement leur rôle et pourquoi elles existent.

Les classes .u-sr-only et .u-sr-only-focusable permettent de rendre du contenu invisible visuellement mais accessible aux lecteurs d’écran. C’est utile pour ajouter des liens « skip link » (sauter au contenu) ou des indications vocales sans encombrer l’interface. .u-not-sr-only permet d’annuler cet effet.

.u-sr-only { ... } .u-sr-only-focusable { ... } .u-not-sr-only { ... }

Naviguer uniquement au clavier est essentiel pour de nombreux utilisateurs. Les classes .u-focus-ring, .u-focus-ring-primary, .u-focus-ring-inset ou .u-focusable garantissent que l’élément reçoit un anneau de focus clair au clavier. À l’inverse, .u-focus-outline-none supprime l’outline, mais doit être utilisé avec précaution car il peut rendre la navigation inaccessible.

Les animations peuvent être gênantes pour certaines personnes. Avec .u-reduce-motion, les transitions et animations sont fortement réduites si l’utilisateur a activé l’option « Réduire les animations » dans son système. La classe .u-no-animations coupe totalement les animations.

Certains environnements (Windows High Contrast, ou réglages prefers-contrast) imposent des couleurs spécifiques. Les classes comme .u-contrast-boost aident à renforcer la lisibilité (ex. soulignement plus épais, bords visibles). De plus, les anneaux de focus sont adaptés automatiquement pour rester visibles.

Sur mobile ou tablette, il n’y a pas toujours de survol (hover). La classe .u-hover-only permet de désactiver des effets de hover dans ces contextes. .u-touch-target garantit que les boutons et liens ont une zone cliquable suffisamment grande (44×44 px minimum).

Les attributs aria-* servent à informer les technologies d’assistance (par ex. aria-busy ou aria-disabled). Les règles CSS de l’usine ajoutent une cohérence visuelle (curseur interdit, opacité réduite, etc.) pour ces états. Enfin, les éléments rendus focusables avec tabindex reçoivent un outline visible.

  • Visibilité : masqué visuellement mais lu par lecteur d’écran (.u-sr-only).
  • Focus : anneaux clairs et visibles au clavier (.u-focus-ring).
  • Mouvement : réduction ou suppression des animations si demandé.
  • Contraste : support des thèmes forcés et contraste élevé.
  • Interactions : cibles tactiles suffisantes, gestion du hover.
  • ARIA : styles visuels alignés avec les attributs d’état.
